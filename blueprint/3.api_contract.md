# Ghostwaiter API Contract v1.2

Base URL untuk semua endpoint adalah `/api` kecuali endpoint statis seperti `/`, `/manifest.webmanifest`, `/service-worker.js`.

## 1. Authentication

### `POST /api/auth/login`
Login dan buat session cookie.

Request body:
```json
{
  "password": ""
}
```

Response:
```json
{
  "status": "success",
  "session_token": "..."
}
```

### `POST /api/auth/logout`
Menghapus cookie session.

### `GET /api/auth/status`
Mengembalikan status auth.

Response:
```json
{
  "authenticated": true,
  "password_required": false
}
```

## 3. Chat APIs

### `POST /api/chat/new`
Buat chat kosong.

Request:
```json
{
  "workspace_id": "writing"
}
```

Response:
```json
{
  "chat_id": "chat_xxx"
}
```

### `GET /api/chat/list?workspace_id=<id>&archived=true|false`
List chat berdasarkan workspace dan status archive.

### `GET /api/chat/session/{chat_id}?workspace_id=<id>`
Ambil detail chat.

### `POST /api/chat/archive`
Archive chat.

Request:
```json
{
  "workspace_id": "writing",
  "chat_id": "chat_xxx"
}
```

### `POST /api/chat/restore`
Restore chat dari archive.

### `POST /api/chat/rename`
Rename judul chat.

### `POST /api/chat/delete-permanent`
Hapus permanen chat yang sudah archived.

### `POST /api/chat/send`
Kirim pesan chat dan stream respons AI.

Request:
```json
{
  "workspace_id": "writing",
  "chat_id": "chat_xxx",
  "message": "Halo"
}
```

Response:
- streaming plain text
- header `X-Chat-Id` berisi chat id

## 6. Brain & learning APIs

### `POST /api/brain/learn/revision`
Menerima AI output + revisi user untuk ekstraksi style/thinking patterns.

Request:
```json
{
  "workspace_id": "writing",
  "ai_output": "...",
  "user_revision": "..."
}
```

Response:
```json
{
  "status": "learned",
  "analysis": {
    "style_rules": [],
    "thinking_patterns": []
  }
}
```

### `POST /api/brain/compare-revision`
Mirip dengan `learn/revision`, tetapi hanya menganalisis tanpa menyimpan.

### `POST /api/brain/commit-revision`
Menyimpan analisis revisi ke profile brain.

### `POST /api/brain/learn/raw-writing`
Menganalisis tulisan asli pengguna.

Request:
```json
{
  "workspace_id": "writing",
  "content": "Tulisan asli",
  "type": "user"
}
```

### `GET /api/brain/profile?workspace_id=<id>`
Ambil semua data brain terkait workspace.

Response:
```json
{
  "style_profile": {},
  "thinking_profile": {},
  "rules": [],
  "memory": [],
  "conversation_memory": [],
  "pending_proposals": 0,
  "revision_count": 0,
  "raw_writing_count": 0
}
```

### `POST /api/brain/item/update`
Update item brain (style, thinking, atau memory).

### `POST /api/brain/item/delete`
Delete item brain.

### `GET /api/brain/proposals?workspace_id=<id>&status=pending|approved|rejected|all`
List proposal pembelajaran.

### `POST /api/brain/proposals/approve`
Approve proposal.

### `POST /api/brain/proposals/reject`
Reject proposal.

### `POST /api/brain/proposals/bulk`
Approve atau reject banyak proposal sekaligus.

Request:
```json
{
  "workspace_id": "writing",
  "action": "approve",
  "proposal_ids": ["proposal_1"]
}
```

## 11. Notes on streaming and auth

- Endpoint chat dan generate memakai `StreamingResponse`.
- Headers `X-OpenRouter-Key`, `X-OpenRouter-Model`, dan `X-AI-Provider` dipakai untuk mengirim credential dan pilihan provider/model yang sudah ditentukan pengguna ke backend.
- `Authorization: Bearer <token>` atau cookie session bisa dipakai untuk auth.

18. WORKFLOW UTAMA

Writing Flow

1. user input prompt
2. AI generate
3. user edit
4. brain.learn(revision)
5. sync ke GitHub

---

Chat Flow

1. user chat
2. AI streaming response
3. auto save session
4. summary update (jika panjang)

Learning Flow

1. detect event (revision / chat / raw writing)
2. extract pattern
3. update:
   - style_profile
   - thinking_profile
   - rules
   - memory

---

Reference Flow (Semi Auto)

1. AI detect need reference
2. ask user permission
3. if yes:
   - search web
   - summarize
   - store in references

19. Update v1.1 (API Changes)

- **Limitless Input**: Tidak ada batasan `max_length` pada endpoint `/chat/send`, `/ai/generate`, `/brain/learn/raw-writing`, dan endpoint terkait lainnya.
- **Error Handling Stream**: API tetap menggunakan _Streaming text chunks_, namun sekarang frontend menangani interupsi stream secara _graceful_ tanpa memunculkan error text palsu.

---

20. Update v1.2 (Data Sync API)

- Endpoint baru: `POST /api/import` menerima payload `UploadFile` (format ZIP hasil `/api/export`).
- Backend memproses ZIP, memeriksa list workspace, menghindari konflik `id`, dan menyimpan file import menjadi workspace independen.

---

21. Update v1.3 (Advanced API Changes)

- **Sync GitHub Refactor**:
  - `POST /sync/push`: Mengunggah status lokal ke GitHub (overwrite) menggunakan GitHub Git Database API (Trees).
  - `POST /sync/pull`: Mengunduh (overwrite) *state* lokal dari repositori GitHub.
- **Brain Bulk Actions**:
  - `POST /brain/proposals/bulk`: Endpoint baru untuk aksi massal. Menerima payload `{"action": "approve" | "reject", "proposal_ids": [...]}`.
- **Brain CRUD APIs**:
  - Endpoint `PUT /brain/item` dan `DELETE /brain/item` (atau metode setara melalui `main.py`) mendukung pengeditan dan penghapusan `style`, `thinking`, dan `memory` secara individual.
- **Model Dynamic Awareness**:
  - Tidak ada lagi model bawaan yang dipaksa oleh server; UI mengirim provider/model yang dipilih pengguna, dan backend hanya meneruskan konfigurasi tersebut ke layanan AI yang sesuai.
