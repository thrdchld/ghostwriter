# Ghostwaiter Data Schema v1.2

Dokumen ini menjelaskan struktur data yang dipakai oleh aplikasi Ghostwaiter v1.2. Semua data utama disimpan sebagai file JSON pada storage lokal dan dapat dibackup ke GitHub melalui API sinkronisasi manual.

## 1. Struktur root storage

Struktur default storage lokal adalah:

```text
data/
├── system/
│   ├── settings.json
│   ├── models.json
│   └── workspaces.json
├── workspaces/
│   └── <workspace_id>/
│       ├── drafts/
│       ├── chats/
│       ├── brain/
│       ├── references/
│       ├── summary/
│       ├── learning/
│       │   ├── revision_pairs/
│       │   ├── raw_writing/
│       │   └── chat_patterns/
│       └── settings/
├── queue/
│   └── pending_sync.json
├── snapshots/
│   ├── manifest.json
│   └── *.zip
└── archive/
```

> `DATA_DIR` dapat dipakai untuk mengubah lokasi root, tetapi struktur internal tetap sama.

## 2. File sistem

### `system/settings.json`
Menampung status aplikasi global.

Contoh:
```json
{
  "schema_version": 1,
  "active_workspace": "writing",
  "theme": "auto",
  "sync_status": "idle",
  "last_sync": ""
}
```

### `system/models.json`
Menampung metadata provider/model yang dipilih pengguna di UI, tanpa mengandalkan model bawaan server.

```json
{
  "schema_version": 1,
  "provider": "openrouter",
  "model": "",
  "last_updated": "2026-06-17T00:00:00+00:00"
}
```

### `system/workspaces.json`
Registry workspace.

```json
{
  "schema_version": 1,
  "default_workspace": "writing",
  "items": [
    {
      "id": "writing",
      "name": "Writing",
      "created_at": "2026-06-17T00:00:00+00:00",
      "updated_at": "2026-06-17T00:00:00+00:00"
    }
  ]
}
```

## 3. Workspace object

Setiap workspace memiliki folder yang sama dengan struktur berikut:

```text
workspaces/<workspace_id>/
├── drafts/
├── chats/
├── brain/
├── references/
├── summary/
└── learning/
```

## 4. Draft entity

File: `workspaces/<workspace_id>/drafts/<draft_id>.json`

Schema umum:
```json
{
  "schema_version": 1,
  "id": "draft_xxx",
  "title": "Untitled",
  "content": "",
  "collections": [],
  "tags": [],
  "created_at": "2026-06-17T00:00:00+00:00",
  "updated_at": "2026-06-17T00:00:00+00:00",
  "status": "active"
}
```

## 5. Chat entity

File: `workspaces/<workspace_id>/chats/<chat_id>.json`

```json
{
  "schema_version": 1,
  "id": "chat_xxx",
  "title": "New Chat",
  "messages": [
    {
      "role": "user",
      "content": "Halo"
    },
    {
      "role": "assistant",
      "content": "Halo juga"
    }
  ],
  "created_at": "2026-06-17T00:00:00+00:00",
  "updated_at": "2026-06-17T00:00:00+00:00",
  "archived": false,
  "summary": "",
  "accessed_workspaces": []
}
```

## 6. Brain files

### `brain/style_profile.json`
Berisi daftar aturan gaya bahasa.

```json
{
  "schema_version": 1,
  "rules": [
    "Gunakan bahasa formal",
    "Gunakan kalimat singkat"
  ]
}
```

### `brain/thinking_profile.json`
Berisi pola pemikiran.

```json
{
  "schema_version": 1,
  "patterns": [
    "Mulai dari masalah utama",
    "Buat struktur argumen"
  ]
}
```

### `brain/memory.json`
Berisi memori permanent.

```json
{
  "schema_version": 1,
  "items": [
    {
      "id": "mem_xxx",
      "content": "Pengguna suka paragraf singkat",
      "created_at": "2026-06-17T00:00:00+00:00"
    }
  ]
}
```

### `brain/rules.json`
Berisi aturan eksplisit tambahan.

```json
{
  "schema_version": 1,
  "max_rules": 100,
  "items": [
    {
      "id": "rule_xxx",
      "content": "Jangan terlalu panjang",
      "created_at": "2026-06-17T00:00:00+00:00"
    }
  ]
}
```

### `brain/conversation_memory.json`
Berisi konsep dari chat yang sudah dianalisis.

```json
{
  "schema_version": 1,
  "items": [
    {
      "id": "concept_xxx",
      "content": "Target audiens mahasiswa",
      "source_chat_id": "chat_xxx",
      "created_at": "2026-06-17T00:00:00+00:00"
    }
  ]
}
```

### `brain/learning_proposals.json`
Daftar usulan yang menunggu persetujuan.

```json
{
  "schema_version": 1,
  "items": [
    {
      "id": "proposal_xxx",
      "type": "style",
      "content": "Gunakan nada lebih hangat",
      "status": "pending",
      "source_chat_id": "chat_xxx",
      "created_at": "2026-06-17T00:00:00+00:00",
      "updated_at": "2026-06-17T00:00:00+00:00"
    }
  ]
}
```

## 7. Reference entity

File: `workspaces/<workspace_id>/references/<reference_id>.json`

```json
{
  "schema_version": 1,
  "id": "ref_xxx",
  "title": "Contoh referensi",

  "url": "https://example.com",
  "summary": "Ringkasan hasil pencarian",
  "tags": ["keyword"],
  "created_at": "2026-06-17T00:00:00+00:00"
}
```

## 8. Learning data

### `learning/revision_pairs`
Setiap file merekam pasangan output AI dan revisi pengguna.

```json
{
  "schema_version": 1,
  "id": "rev_xxx",
  "ai_output": "...",
  "user_revision": "...",
  "analysis": {
    "style_rules": [],
    "thinking_patterns": []
  },
  "created_at": "2026-06-17T00:00:00+00:00"
}
```

### `learning/raw_writing`
Menampung contoh tulisan pengguna.

```json
{
  "schema_version": 1,
  "id": "raw_xxx",
  "content": "Tulisan asli",
  "type": "user",
  "analysis": "Aturan gaya yang dihasilkan AI",
  "created_at": "2026-06-17T00:00:00+00:00"
}
```

## 9. Queue and snapshots

### `queue/pending_sync.json`
Queue perubahan yang belum dipush ke GitHub.

```json
{
  "schema_version": 1,
  "items": [
    {
      "id": "sync_xxx",
      "type": "drafts",
      "workspace": "writing",
      "payload": {
        "id": "draft_xxx",
        "updated_at": "2026-06-17T00:00:00+00:00"
      },
      "created_at": "2026-06-17T00:00:00+00:00"
    }
  ]
}
```

### `snapshots/manifest.json`
Manifest snapshot ZIP yang dibuat dari data lokal.

```json
{
  "schema_version": 1,
  "items": [
    {
      "id": "snapshot_20260617_120000",
      "file": "snapshot_20260617_120000.zip",
      "created_at": "2026-06-17T00:00:00+00:00",
      "workspace_count": 1,
      "size": 12345
    }
  ]
}
```

## 10. Sinkronisasi dan integritas

- Semua file JSON yang berubah akan menambah entri ke queue sync.
- Sinkronisasi GitHub bersifat manual dan penuh overwrite pada data repo target.
- `schema_version` wajib ada pada setiap file yang menggunakan format yang didukung aplikasi.
