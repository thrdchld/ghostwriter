# Ghostwaiter UI Wireframe v1.2

## 1. Prinsip UI

- Mobile-first dengan pengalaman yang tetap jelas pada layar kecil.
- Fokus pada alur menulis, berdiskusi, dan melatih AI.
- Komponen modal dan toast dipakai untuk feedback yang ringkas.
- Tidak ada animasi berat; interaksi harus tetap cepat.

## 2. Layout global

Aplikasi memakai layout single-page dengan area utama yang berubah sesuai navigasi.

```text
[ Sidebar / top nav ]
[ Topbar status cluster ]
[ Main content view ]
[ Modal / sheet / toast ]
```

### Topbar
- kiri: branding aplikasi Ghostwaiter (desktop) + tombol buka sidebar pada mobile
- kanan: workspace pill, model status pill, sync status pill

### Navigasi utama
- Desktop: sidebar vertikal dengan tombol Chat, Write, Brain, Settings.
- Mobile: tombol navigasi tetap terlihat dalam layout yang compact, dengan toggle sidebar.

## 3. View: Chat

```text
[ Chat header ]
  title + history button + new chat button
[ chat messages ]
  bubble user / assistant
  copy button on assistant bubble
[ composer ]
  textarea + send button
```

Fitur:
- streaming balasan AI
- auto scroll saat token masuk
- render Markdown aman
- tombol rename, archive, restore, delete permanent pada riwayat chat

## 4. View: Write

```text
[ section heading ]
  Meja Tulis + Draft button
[ prompt card ]
  textarea prompt
  mode chips: Tulis / Tulis Ulang / Parafrase
  Generate button
[ editor card ]
  title input
  textarea editor
  footer: save-state, word count, copy/train buttons
```

Fitur:
- generate dari prompt
- draft autosave lokal
- word count + status save
- tombol train untuk mengirim revisi ke Brain

## 5. View: Brain

```text
[ section heading ]
  Brain + refresh button
[ stats cards ]
  style / thinking / revision count
[ brain tabs ]
  Teach | Compare | Style | Thinking | References | Memory | Proposals
[ content panel ]
  depends on selected tab
[ bulk proposal actions ]
  visible when proposal exists
```

Tab utama:
- Teach: input raw writing
- Compare: compare original vs edited
- Style: daftar gaya bahasa
- Thinking: daftar pola berpikir
- References: hasil search web
- Memory: memori personal
- Proposals: review usulan AI

## 6. View: Settings

```text
[ heading ]
[ list settings ]
  Tema
  Provider AI + API key + Load model
  Sinkronisasi GitHub
  Impor Data Offline
  Ekspor Data Offline
  Instal Aplikasi
  Keluar
[ footer version ]
```

## 7. Modal dan sheet

### Confirm modal
Digunakan untuk aksi destruktif seperti archive, delete, restore, atau sinkronisasi.

### Prompt modal
Digunakan untuk input teks seperti rename workspace / rename chat.

### Sync modal
Digunakan saat tombol GitHub sync ditekan. Terdiri dari:
- `Kirim ke GitHub (Push)`
- `Ambil dari GitHub (Pull)`
- `Batal`

### Sheet panel
Dipakai untuk daftar chat, daftar workspace, dan opsi navigasi tambahan.

## 8. Feedback UI

- Toast untuk success/error/info.
- Status pill menampilkan kondisi model dan sinkronisasi.
- Fokus outline dan tombol yang sesuai untuk keyboard navigation.

## 9. Responsive behavior

### Desktop
- sidebar kiri tetap visible
- area konten utama penuh

### Tablet / mobile
- sidebar bisa ditutup
- topbar dan area konten tetap ringkas
- interaksi harus menggunakan target yang cukup besar untuk tap

## 10. Core user flows

### Chat flow
`message → stream → save chat → update summary/proposals`

### Write flow
`prompt → generate → edit → save draft → train revision`

### Brain flow
`raw text / compare / chat analysis → proposal → approve/reject → update brain profile`

## 11. UI state persistence

Aplikasi menyimpan beberapa state di localStorage, termasuk:
- view aktif
- tema
- sidebar state
- session token
- provider/model preference
