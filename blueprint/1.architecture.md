# Ghostwaiter Architecture v1.2

## 1. Visi produk

Ghostwaiter adalah personal writing assistant yang menggabungkan:

- chat percakapan untuk brainstorming dan feedback,
- editor tulisan untuk draft dan rewrite,
- sistem pembelajaran personal untuk style, thinking, memory, dan rules,
- sinkronisasi manual ke GitHub untuk backup dan restore.

Tujuan utamanya bukan sekadar menjawab pertanyaan, melainkan membantu pengguna menghasilkan dan mempertahankan kualitas tulisan secara konsisten.

## 2. Prinsip desain

- Mobile-first dan ringan.
- Satu halaman (single-page) dengan navigasi cepat.
- Lokal-first untuk draft dan chat, dengan backup opsional.
- Per-workspace isolation untuk konteks yang berbeda.
- Tidak ada terminal atau command execution dalam aplikasi.

## 3. Arsitektur runtime

### Frontend
- HTML, CSS, dan vanilla JavaScript.
- PWA dengan service worker dan manifest.
- UI terdiri dari empat area utama: Chat, Write, Brain, dan Settings.

### Backend
- FastAPI untuk REST API dan file serving.
- Endpoint streaming untuk chat dan generate.
- Session auth via cookie atau bearer token.

### AI layer
- Provider abstraction melalui HTTP API.
- Provider yang didukung: OpenRouter, Google Gemini, Groq, DeepSeek, Mistral, Kilo.
- Model dipilih dari UI, dan header provider/model dikirim bersama request.

## 4. Alur data utama

1. User membuka app dan backend memvalidasi session.
2. Frontend memuat current workspace, sync status, dan model status.
3. Chat dan generate tulisan mengirim request ke backend.
4. Backend membangun context dari workspace brain data.
5. Model output streamed ke frontend dan disimpan ke storage.
6. User dapat mengirim hasil ke Brain untuk learning / proposal.

## 5. Workspace model

Setiap workspace bersifat terisolasi:

- `writing` adalah workspace default yang tidak boleh dihapus.
- Workspace lain dapat dibuat, diubah nama, atau dihapus.
- Brain data, chat, draft, references, dan summary masing-masing disimpan terpisah per workspace.

## 6. Brain system

Ghostwaiter menggunakan beberapa lapisan personalisasi:

- Style rules: aturan gaya bahasa yang muncul dari revisi dan raw writing.
- Thinking patterns: pola cara berpikir dan struktur argumen.
- Memory: fakta/favorit/konvensi yang perlu diingat.
- Rules: constraint khusus yang harus dipatuhi AI.
- Conversation memory: konsep yang dipetik dari chat.
- Learning proposals: usulan yang menunggu approve/reject.

## 7. Storage system

Data utama disimpan sebagai file JSON, bukan satu database tunggal.

- Root storage bisa diatur lewat `DATA_DIR`.
- Jika `/data` tersedia dan writable, aplikasi akan memakai `/data/ghostwaiter`.
- Semua perubahan data menambahkan entri ke queue sync sehingga backup bisa dilakukan manual.

## 8. Sync & backup

Sinkronisasi dilakukan secara manual dari UI:

- Push: menulis isi data lokal ke repo GitHub.
- Pull: membaca data dari repo GitHub dan menimpa data lokal.

Proses ini dibuat manual untuk menjaga integritas data dan menghindari konflik yang tidak terkontrol.

## 9. Offline & resilience

Saat offline:

- draft dan UI masih dapat digunakan,
- data lokal yang sudah ada tetap bisa dibaca,
- export/import dan snapshot tetap menjadi mekanisme pemulihan.

## 10. Security & operational notes

- API keys tidak dipublikasikan ke frontend kecuali lewat request header yang sesuai.
- Session secret dan password dapat dipakai untuk proteksi aplikasi.
- Tidak ada command execution dari UI atau backend.

## 11. Versioning note

Dokumen ini menyesuaikan dengan versi aplikasi Ghostwaiter v1.2.

## 12. End-to-end flow summary

Chat flow:
- user message → store chat → stream AI response → save message → update summary/proposals.

Write flow:
- prompt → stream generated text → save/edit draft → train / learn from revision.

Learning flow:
- compare / raw input / chat analysis → generate proposals → approve or reject → update brain profiles.
